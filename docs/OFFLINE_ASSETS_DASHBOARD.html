Desktop Offline-First Asset & Export Pipeline

Complete technical specification for running the Printpoz photo editor locally — covering every asset type, every API endpoint, every data flow.

Core principle: everything the user interacts with on the canvas runs locally. User photos are never uploaded. Library assets (backgrounds, stickers, masks) are fetched from the API on first use and cached on disk. Fonts from the backend are cached as WOFF2 files. Google Fonts and AI features remain online. Export renders entirely on-device.

Native Storage & Sync Stack

The application operates on an offline-first data model using local-first storage technologies:

  • SQLite Storage: Local index, version history (delta-snapshots), and sync queue run via better-sqlite3.
  • Content-Addressable File Store: Files (user photos, stickers, masks) are hashed via SHA-256 for automatic binary deduplication.
  • Atomic Auto-Saves & Backups: Writes to temporary files and renames to prevent corruption; local rolling backup ring.
  • Durable Background Sync: Replays pending cloud operations with exponential backoff on reconnection (mobile-parity).
  • LWW Conflict Handling: Last-Write-Wins resolution with user prompts for concurrent cloud modifications.
  • OS Keychain Security: Sensitive session tokens encrypted via system keychain (via keytar/safeStorage).

Advanced Desktop-Only Advantages

Electron architecture unlocks capabilities that web browsers restrict:

  • Extreme Performance: Stream photos directly from disk (zero-upload lag); multi-threaded background decoders; bypass browser RAM limits.
  • Native OS Integration: Drag-and-drop folders directly to canvas; double-click .ppzproject to open; access installed system fonts.
  • Print & Color Dominance: True 300+ DPI local rendering; CMYK & ICC color management; local soft-proofing and spool printing.
  • Enhanced Productivity: Sync projects in the background after editor close; set "watch-folders" for auto-import; silent auto-updates.
  • Offline Capability: Run editing (transforms, filters, layouts), undo/redo, local save/load, and PDF compilation 100% offline.

3. Architecture

2.1 User Photo Pipeline (No Upload)

  User selects image file
    │
    ├── [Web] File → startEagerResize() → uploadImageOptimized()
    │         → S3 init → S3 PUT chunks → S3 complete → https:// URL
    │
    └── [Desktop] File.path → startEagerResize() → IPC fs:saveLocalAsset
                  → copyFile to AppData/projects/<id>/assets/<uuid>.ext
                  → app-assets://project/<id>/<uuid>.ext

Key insight: Electron's File objects include a .path property with the absolute filesystem path. This is how we bypass the entire upload pipeline — we copy the file directly using Node's fs.copyFile().

The eager resize for canvas preview thumbnails still runs identically on both platforms. Only the "upload" step is replaced with a local file copy.

2.2 Library Asset Cache Pipeline

  User opens Backgrounds/Stickers/Masks panel
    │
    ├── API call to fetch list metadata (always online)
    │   → Panel renders thumbnails from CDN URLs
    │
    └── User clicks asset to place on canvas
          │
          ├── desktop.cache.check(category, assetId)
          │     │
          │     ├── [Cached] → app-assets://cache/<category>/<assetId>
          │     │                (instant, zero network)
          │     │
          │     └── [Not cached]
          │           │
          │           ├── [Online] → desktop.cache.download(url, category, assetId)
          │           │               → writes to AppData/cache/<category>/
          │           │               → returns app-assets:// URL
          │           │
          │           └── [Offline] → falls back to remote URL
          │                           (broken image indicator)

2.3 Export Pipeline (Fully Client-Side)

  Export triggered → svgData.svgContent[] ready
    │
    ├── [Web] For each page:
    │     apiPost(ENDPOINTS.exportAsJPG, { svgDetails, fonts, w, h })
    │     → Server headless Chrome renders → returns JPEG blob
    │     → jsPDF compiles → saveAs() downloads to browser
    │
    └── [Desktop] For each page:
          IPC export:renderSvg(svgContent, width, height)
          → Main process spawns hidden BrowserWindow (offscreen: true)
          → Loads SVG + @font-face rules pointing to cached WOFF2 files
          → document.fonts.ready → webContents.capturePage()
          → Returns JPEG buffer → win.close()
          → jsPDF compiles → desktop.dialog.saveAs()
          → desktop.fs.writeFile() to user-chosen path

4. Code Implementation

// electron/main/protocol.ts — Dual-scope asset protocol handler
protocol.handle("app-assets", async (request) => {
  const url  = new URL(request.url);
  const scope = url.host;                     // "project" | "cache"
  const parts = decodeURIComponent(url.pathname).split("/").filter(Boolean);
  const userData = app.getPath("userData");

  let baseDir: string;
  if (scope === "project") {
    // app-assets://project/<projectId>/<filename>
    baseDir = join(userData, "projects", parts[0], "assets");
  } else if (scope === "cache") {
    // app-assets://cache/<category>/<filename>
    baseDir = join(userData, "cache", parts[0]);
  } else {
    return new Response("Bad scope", { status: 400 });
  }

  const filename = parts.slice(1).join("/");
  const filePath = normalize(join(baseDir, filename));

  // Path traversal guard
  if (!filePath.startsWith(normalize(baseDir) + sep)) {
    return new Response("Forbidden", { status: 403 });
  }

  try {
    return await net.fetch(pathToFileURL(filePath).toString());
  } catch {
    return new Response("Not found", { status: 404 });
  }
});
// electron/main/ipc/assets.ipc.ts — Local file copy for user photos
ipcMain.handle("fs:saveLocalAsset", async (_e, filePath, projectId) => {
  if (!filePath || !projectId) {
    return { ok: false, error: "Missing filePath or projectId" };
  }

  const assetsDir = join(app.getPath("userData"), "projects", projectId, "assets");
  await mkdir(assetsDir, { recursive: true });

  // Preserve extension, generate unique name
  const ext = extname(filePath);
  const uniqueName = `${uuidv4()}${ext}`;
  const destPath = join(assetsDir, uniqueName);

  await copyFile(filePath, destPath);

  return {
    ok: true,
    data: {
      url: `app-assets://project/${projectId}/${uniqueName}`,
      filename: uniqueName,
    },
  };
});
// electron/main/ipc/cache.ipc.ts — On-demand library asset caching
ipcMain.handle("cache:check", async (_, category, assetId) => {
  const filePath = join(app.getPath("userData"), "cache", category, assetId);
  return existsSync(filePath);
});

ipcMain.handle("cache:download", async (_, remoteUrl, category, assetId) => {
  const dir = join(app.getPath("userData"), "cache", category);
  await mkdir(dir, { recursive: true });

  // Stream the file from the CDN
  const response = await net.fetch(remoteUrl);
  if (!response.ok) {
    return { ok: false, error: `HTTP ${response.status}` };
  }

  const buffer = Buffer.from(await response.arrayBuffer());
  await writeFile(join(dir, assetId), buffer);

  return {
    ok: true,
    data: { url: `app-assets://cache/${category}/${assetId}` },
  };
});
// electron/main/services/exporter.ts — Offscreen SVG → JPEG renderer
export async function renderSvgToBuffer(
  svgContent: string,
  width: number,
  height: number,
  fontFaceCSS: string = ""
): Promise<Buffer> {
  const win = new BrowserWindow({
    width: Math.ceil(width),
    height: Math.ceil(height),
    show: false,
    webPreferences: { offscreen: true },
  });

  const html = `<!DOCTYPE html>
    <html><head><style>
      ${fontFaceCSS}
      body,html{margin:0;padding:0;overflow:hidden}
      svg{width:100%;height:100%;display:block}
    </style></head>
    <body>${svgContent}</body></html>`;

  await win.loadURL(
    `data:text/html;charset=utf-8,${encodeURIComponent(html)}`
  );
  await win.webContents.executeJavaScript("document.fonts.ready");

  const nativeImage = await win.webContents.capturePage();
  win.close();
  return nativeImage.toJPEG(95);
}

7. Edge Cases

User offline, opens Backgrounds panelAPI call fails → panel shows empty state or last-known list from in-memory cache. Previously cached assets still render on canvas.
User offline, places already-cached sticker✅ Works — served from app-assets://cache/
User offline, places uncached sticker❌ Fails — download fails, remote URL used, shows broken image. Non-blocking error toast.

Cache exceeds 500 MBLRU eviction removes least-recently-accessed files. Project-scoped assets are never evicted.
