Blank Theme — Random-Layout Generator

Session notes — 2026-07-08 · a "Blank design" flow for photobook & layflat that builds N spreads, each filled with a random, non-repeating layout in a chosen images-per-spread range. Boxes stay empty. Reuses the Shuffle reducers for placement; fetches the layouts catalog once then runs from a local cache.

🏛️ Architecture → ⬆️ Upload Pipeline → 🅿️ Photobook Full Cover Sync → 📏 Canvas Rulers → 📦 Offline Assets → 🎲 Blank Theme Generator (2026-07-08) — you are here
Scope: a generator (generateRandomLayoutPages) shared by two entry points — the design-selection Blank design card and the in-editor Themes-tab card. It builds empty pages then drives the same layout-apply reducers the Layout sidebar's Shuffle uses, so it inherits their proven scaling/bleed math. Photobook / layflat only.

Spread vs single — one shared usedIds set for variety

Each page index is classified (mirrors the Layout sidebar). Covers are single; photobook interior is a 2-up spread. Every pick avoids a layout already used.

page 2 · interior

page 3 · interior

The core function

// src/library/utils/helpers/blankThemeGenerator.js
export async function generateRandomLayoutPages(dispatch, { pageCount, minImages, maxImages }) {
  const { editorType, canvasSize, settings } = store.getState().canvas.present;

  const usesHalfWidthLayout = editorType === PHOTOBOOK || settings.isFoldable;
  const splitby   = usesHalfWidthLayout ? 2 : 1;               // MUST match the sidebar
  const fullWidth = canvasSize.width, height = canvasSize.height;

  const spreadFlags = range(total).map(i => isSpreadIndex(editorType, settings, i, total));
  const [spreadPool, pagePool] = await Promise.all([          // cache-first, scaled, grouped
    needSpread ? fetchLayoutPool(2, fullWidth, height, splitby) : {},
    needSingle ? fetchLayoutPool(1, fullWidth, height, splitby) : {},
  ]);

  dispatch(applyTheme(range(total).map(makeBlankPage)));       // pages with layout: []

  const usedIds = new Set();                                   // no repeats
  for (let i = 0; i < total; i++) {
    const target = randInt(minImages, maxImages);
    dispatch(setPageNumber(i));
    if (spreadFlags[i]) dispatch(setEntireSpreadLayout(pickLayout(spreadPool, target, usedIds)));
    else { dispatch(setActiveSide(0)); dispatch(setPageLayout(pickLayout(pagePool, target, usedIds)[0])); }
  }
  return clone(store.getState().canvas.present.pages);         // built pages, for persisting
}

Three bugs it hit — and why

❌ Crash: reading 'some' of undefined

replaceSettings({subtype:""}) wiped settings.selectedMenuItems; the customer Sidebar does settings.selectedMenuItems.some(...) unguarded.

✅ Fix

Use setSettings (merge), not replace — keeps selectedMenuItems. And guard the Sidebar read (fall back to the full menu).

❌ Footer stuck loading

Footer gates on projectSetup.isThemeApplied. The customer create path navigates restore=1 without staging a snapshot, so useEditorSnapshot's setThemeApplied(true) never fired.

✅ Fix

Dispatch setThemeApplied(true) in the customer create path — a design is applied.

❌ Same layout repeated

pickLayout had no memory of what it already placed.

✅ Fix

Shared usedIds set; prefer unused layouts, nearest-count fallback, repeat only when exhausted.

Porting recipe (any app / theme / website)

Framework-agnostic — you need four capabilities:

CapabilityWhat it is · what to reuse
Layouts catalogList of templates, each with an image count + boxes in some reference space. Filterable by image count and single-vs-spread.
Layout → canvas placerReuse your editor's existing "apply a layout to a page". Don't reimplement the geometry. (Here: setPageLayout / setEntireSpreadLayout.)
Page modelCreate N empty pages; apply a layout to page i. (Here: applyTheme + setPageNumber.)
CacheAny KV store, memory + durable. Store the raw catalog under a stable key; read before fetching.
function generate(pageCount, minImages, maxImages) {
  const layouts = cacheFirstFetch();            // memory → durable → API
  const used = new Set();
  const pages = [];
  for (let i = 0; i < pageCount; i++) {
    const target   = randInt(minImages, maxImages);
    const template = pickUnused(layouts, target, used);   // nearest-count fallback
    const page     = newBlankPage();
    applyLayout(page, template, targetPageSize);          // REUSE your placer
    pages.push(page);
  }
  return pages;
}

Gotchas checklist

  • ✔ Set editorType + canvasSize (+ clean settings) before generating — reducers read the store synchronously.
  • ✔ splitby must match the Layout sidebar (photobook/foldable = 2, layflat = 1).
  • ✔ isSpreadIndex must match the sidebar's spread eligibility.
  • ✔ Merge settings, never replace (keeps selectedMenuItems etc.).
  • ✔ Set isThemeApplied (or your "ready" flag) after applying pages.
  • ✔ clearHistory() after generation (it floods the undo stack).
  • ✔ Cache the raw catalog; only fetch when nothing is stored locally.
  • ✔ Track usedIds; nearest-count fallback, then repeat.
