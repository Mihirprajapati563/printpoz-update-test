Offline Assets Spec Export Pipeline Spec

Desktop Export Pipeline

SVG → JPEG rendering moves from server-side headless Chrome to a local Electron offscreen BrowserWindow. Zero network traffic. 5–10× faster. Fully offline.

1. The Problem

The current export pipeline sends each page's SVG to the server API (POST /cart-editor-details/exportAsJpeg), which renders it via headless Chrome and returns a JPEG blob. For a 10-page photobook, this means 10 sequential HTTP round trips — each carrying a multi-megabyte SVG payload up and receiving a multi-megabyte JPEG down.

2. How It Works

Web (Current)
Canvas pages → svgData.svgContent[i]
  → apiPost(exportAsJPG, { svgDetails, fonts, w, h })
    → POST to server → headless Chrome render → JPEG blob response
  → jsPDF.addImage(blob)
  → saveAs(pdfBlob, "file.pdf")  // browser Downloads
Desktop (Replacement)
Canvas pages → svgData.svgContent[i]
  → IPC "export:renderSvg" (svgContent, width, height)
    → Main process: hidden BrowserWindow (offscreen: true)
    → Load SVG + @font-face rules (cached WOFF2 files)
    → document.fonts.ready → capturePage() → toJPEG(95)
    → Return Buffer via IPC
  → jsPDF.addImage(buffer)
  → desktop.dialog.saveAs()
    → desktop.fs.writeFile(chosenPath)  // any folder

4. Implementation Code

// electron/main/services/exporter.ts
import { BrowserWindow } from "electron";

export async function renderSvgToJpeg(
  svgContent: string,
  width: number,
  height: number,
  fontFaceCSS: string = ""
): Promise<Buffer> {
  const win = new BrowserWindow({
    width: Math.ceil(width),
    height: Math.ceil(height),
    show: false,
    webPreferences: { offscreen: true },
  });

  const html = `<!DOCTYPE html>
    <html><head><style>
      ${fontFaceCSS}
      *{margin:0;padding:0;box-sizing:border-box}
      body{overflow:hidden}
    </style></head>
    <body>${svgContent}</body></html>`;

  await win.loadURL(
    \`data:text/html;charset=utf-8,\${encodeURIComponent(html)}\`
  );

  // Ensure all @font-face declarations finish loading
  await win.webContents.executeJavaScript("document.fonts.ready");

  // Allow a paint frame to commit
  await new Promise(r => setTimeout(r, 50));

  const image = await win.webContents.capturePage();
  win.close();

  return image.toJPEG(95);  // 95% quality
}
// electron/main/ipc/export.ipc.ts
import { ipcMain } from "electron";
import { renderSvgToJpeg } from "../services/exporter";
import { buildFontFaceCSS } from "../services/fontResolver";

export function registerExportIPC() {
  ipcMain.handle(
    "export:renderSvg",
    async (_event, svgContent, width, height, fontRefs) => {
      // Build @font-face CSS from cached WOFF2 files
      const fontCSS = fontRefs?.length ? buildFontFaceCSS(fontRefs) : "";

      const jpegBuffer = await renderSvgToJpeg(
        svgContent, width, height, fontCSS
      );

      // Return as ArrayBuffer for IPC transport
      return jpegBuffer.buffer.slice(
        jpegBuffer.byteOffset,
        jpegBuffer.byteOffset + jpegBuffer.byteLength
      );
    }
  );
}
// Inside useExportPages.js — wrapping each exportAsJPG call
import { isDesktop, desktop } from "../../desktop";

async function renderPage(svgContent, width, height, fontRefs) {
  if (isDesktop) {
    // Route through local Electron renderer
    const arrayBuffer = await desktop.export.renderSvg(
      svgContent, width, height, fontRefs
    );
    return new Blob([arrayBuffer], { type: "image/jpeg" });
  }

  // Web fallback — server-side render
  return await apiPost(ENDPOINTS.exportAsJPG, {
    svgDetails: svgContent,
    w: width,
    h: height,
  }, { responseType: "blob" });
}

// Usage (replacing every apiPost(ENDPOINTS.exportAsJPG, ...) call):
const jpegBlob = await renderPage(
  imageData.svgDetails,
  imageData.w,
  imageData.h,
  designFontRefs
);
// Replace saveAs() with native dialog on desktop

async function saveFile(blob, defaultFilename, filters) {
  if (isDesktop) {
    const result = await desktop.dialog.saveAs({
      title: "Save Export",
      defaultPath: defaultFilename,
      filters: filters || [
        { name: "PDF", extensions: ["pdf"] },
        { name: "JPEG", extensions: ["jpg", "jpeg"] },
      ],
    });

    if (result.filePath) {
      const buffer = await blob.arrayBuffer();
      await desktop.fs.writeFile(result.filePath, buffer);
      return true;  // saved
    }
    return false;  // cancelled
  }

  // Web fallback
  saveAs(blob, defaultFilename);
  return true;
}

// Usage:
await saveFile(pdfBlob, `${timestamp}.pdf`, [
  { name: "PDF Files", extensions: ["pdf"] },
]);

5. Edge Cases & Mitigations

ScenarioImpactMitigation
Font not cached at export time Text renders in system fallback font loadDesignFonts() pre-caches all fonts when project loads. Export should rarely encounter missing fonts.
Very large canvas (6000×4000+) High GPU memory during capturePage() Process pages sequentially (not in parallel). One hidden window at a time.
SVG contains <image> with remote URL Image loads from network in hidden window User photos use app-assets:// — no network. Only uncached library assets would hit this.
Export cancelled mid-batch Hidden BrowserWindow may be orphaned Track window reference. On cancel, call win.close() immediately. Use AbortController pattern.
High DPI / Retina display capturePage() may capture at 2× Explicitly set scaleFactor: 1 in capturePage() options for consistent output size.
