1. The architecture (two independent merge/split sites)

Full Cover merges the separate front cover (page 0) and back cover (last page) into one spread on page 0. The logic lives in two places operating on independent data with independent guards:

SiteOperates onOld guard
setSettings reducer — canvas.jslive state.pagessettings transition + data shape (alreadyMerged)
applySpineSettings eager-prop — SizeSettingsPopup.jsxeach saved size's pages_c in allThemesdata shape only (merge/splitPhotobookCover early-return on layout.length)

Size-load paths (selectAndCloseSavedSize, addSize) call setEditorPages/applyTheme (no photobook merge) then setSettings(theme.settings). So all reconciliation must happen in setSettings — the common tail of every load.

2. Root cause

Three sources can leave un-merged pages + showFullCoverSheet: true:

SourceWhat it does
Eager-prop early return (SizeSettingsPopup.jsx)No decodable pages → returns { ...theme, settings: nextSettings }, flipping the flag true while leaving pages_c un-merged.
Transition-gated reducer mergeSwitching from an already-ON size (wasFullCover = true) into an un-merged size makes !wasFullCover false → the merge never runs.
DPI/size convert (confirmMultiConvertDpi)Mints a size with live-merged pages_c but copies sourceTheme.settings, which can disagree.

In all three, page 0 has a single layout[0] (back-cover content) while the renderer believes full cover is on (hideLastCover hides the trailing back page) — back cover on page one, front cover missing.

3. The fix (single choke point)

In setSettings (photobook branch) — src/store/slices/canvas.js:

// RESULTING (effective) full-cover state — NOT a transition flag. Omitted flag
// (partial settings edits) falls back to the existing setting = "unchanged".
const willBeFullCover =
  (dataObj.showFullCoverSheet ?? state.settings?.showFullCoverSheet) === true;
const alreadyMerged = (state.pages[0]?.layout?.length || 0) >= 2;

// ON: merge front + back into a spread on page 0 when the setting is ON
// but the pages aren't merged yet.
if (willBeFullCover && !alreadyMerged && state.pages.length >= 2) { /* merge */ }

// OFF: split the merged spread back into separate front/back pages when the
// setting is OFF but the pages are still merged.
if (!willBeFullCover && alreadyMerged && state.pages.length >= 2) { /* split */ }

What changed: the merge dropped the !wasFullCover transition term; the split dropped the wasFullCover && ...=== false transition terms; willBeFullCover is now the effective value via ??. Kept intact: the [backLayout, frontLayout] orientation, the halfWidth shift, the alreadyMerged guard, the hideLastCover forced-exit, and the spine-width shift.

4. Why partial payloads are safe the landmine avoided

Many setSettings callers send partial payloads. The settings sidebar dispatches setSettings({ ...settings, [key]: value }), which spreads the current settings, so showFullCoverSheet is present with its current value (= the effective value). Truly-omitted payloads fall back to state.settings.showFullCoverSheet via ??.

If omission were read as false, a data-shape split would rip apart an already-merged cover on every unrelated settings edit. The ?? fallback is what makes the data-shape approach safe. The only callers setting a dynamic boolean (applySpineSettings, CreateThemeDialog save payload) always send an explicit true/false, where false unambiguously means "turn off."

6. Scope / what was NOT touched

  • Layflat cover logic — untouched; still gates merge/split on coverEnabled/showFullCoverSheet transitions (its cover model reconstructs the pages array differently).
  • Theme switching (ThemesAction → replaceSettings) — a different flow that fully replaces pages; not part of the reported triggers. replaceSettings does state.settings = payload with no merge/split.
  • applyTheme / setEditorPages — deliberately left without reconcile; they run before setSettings, so they'd act on stale flags.

7. Gotchas (do not regress)

  • Effective value, not raw payload. Always compute willBeFullCover as (dataObj.showFullCoverSheet ?? state.settings.showFullCoverSheet) === true. Reading a missing flag as false splits merged covers on unrelated edits.
  • Reconcile lives in setSettings only. Don't move it into applyTheme/setEditorPages (they run first, before the flag is set).
  • Keep [backLayout, frontLayout] orientation. layout[0] = back (left), layout[1] = front (shifted +halfWidth). Swapping inverts the spread.
  • alreadyMerged keys on state.pages[0].layout.length >= 2. Page 0 is the cover (a single half-sheet when not full-cover); interior spreads also have 2 layouts but are never page 0.
  • setSettings is in ignoredCanvasActions (not recorded in undo). The merge/split mutates state.pages as part of the present state — same as the pre-existing code — so no separate undo step is created.

8. Verification checklist

  • Enable Full Cover → page 0 shows back (left) + front (right) every time.
  • Toggle Full Cover OFF → front and back split into separate pages, content intact.
  • Enable Full Cover, then switch between saved sizes repeatedly → every size shows back-left + front-right (no "back only" page 0).
  • Toggle ON/OFF repeatedly from the Size Settings popup → no drift, no corruption.
  • Turning "Hide Back Cover" off while full cover is on exits full cover and restores the separate back cover page.
  • An unrelated settings change while full cover is on does not split the cover.
  • Spine width still shifts the right-half (front) cover objects correctly when paper thickness changes.
